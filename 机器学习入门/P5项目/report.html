编程练习3：定义衡量标准¶

如果不能对模型的训练和测试的表现进行量化地评估，我们就很难衡量模型的好坏。通常我们会定义一些衡量标准，这些标准可以通过对某些误差或者拟合程度的计算来得到。在这个项目中，你将通过运算决定系数 R2 来量化模型的表现。模型的决定系数是回归分析中十分常用的统计信息，经常被当作衡量模型预测能力好坏的标准。

R2的数值范围从0至1，表示目标变量的预测值和实际值之间的相关程度平方的百分比。一个模型的R2 值为0还不如直接用平均值来预测效果好；而一个R2 值为1的模型则可以对目标变量进行完美的预测。从0至1之间的数值，则表示该模型中目标变量中有百分之多少能够用特征来解释。模型也可能出现负值的R2，这种情况下模型所做预测有时会比直接计算目标变量的平均值差很多。

1、K折交叉验证法 K折交叉验证法是指在数据量不够充足，简单地将数据分割为训练-交叉验证-测试集三部分会导致每一部分数据量不足以得到可信结果的情况下，将数据分成K组，第1次选第1组作为交叉验证集，其它组作为训练集；第2次选第2组作为交叉验证集；依此类推，重复K次。最后获取每个模型K次训练的结果，取平均值最好的训练结果，作为最后选中的模型； 2、GridSearchCV函数在有多个超参数的情况下，允许使用者输入每种超参数的可选值系列，并产生超参数组的所有可选值组合，用每一种超参数组合进行K折交叉验证，并获取验证效果。最终选中效果最好的超参数组合作为结果。 3、cv_results参数包含的信息： 1）各种超参数组合； 2）每种超参数组合在每一种K折数据划分下的训练数据得分和交叉验证数据得分； 3）每种超参数组合在所有数据划分下的平均得分和得分标准差； 4、不使用交叉验证会有以下问题：数据量偏少时，用于交叉验证的数据量太小，交叉验证效果不佳；一部分数据被用于交叉验证，导致训练数据损失了一些信息，训练效果也不佳；使用交叉验证后，能够吸收全部数据用于训练和交叉验证，从而确保不会因为数据分配偏差影响模型选择和训练效果

编程练习 4：训练最优模型¶

在这个练习中，你将需要将所学到的内容整合，使用决策树算法训练一个模型。为了得出的是一个最优模型，你需要使用网格搜索法训练模型，以找到最佳的 'max_depth' 参数。你可以把'max_depth' 参数理解为决策树算法在做出预测前，允许其对数据提出问题的数量。决策树是监督学习算法中的一种。

  4. 定义 'scoring_fnc' 变量: 使用 sklearn.metrics 中的 make_scorer 创建一个评分函数； 将 ‘performance_metric’ 作为参数传至这个函数中；
  5. 定义 'grid' 变量: 使用 sklearn.model_selection 中的 GridSearchCV 创建一个网格搜索对象；将变量'regressor', 'params', 'scoring_fnc'和 'cross_validator' 作为参数传至这个对象构造函数中；

from sklearn.model_selection import KFold
from sklearn.tree import DecisionTreeClassifier
from sklearn.metrics import make_scorer
from sklearn.model_selection import GridSearchCV
def fit_model(X, y):
    """ 基于输入数据 [X,y]，利于网格搜索找到最优的决策树模型"""
    
    cross_validator = KFold(n_splits = 15);
    
    regressor = DecisionTreeClassifier();

    params = {'max_depth':list(range(1,10))};

    scoring_fnc = make_scorer(performance_metric);

    grid = GridSearchCV(estimator=regressor,param_grid=params,scoring=scoring_fnc,cv=cross_validator);

    # 基于输入数据 [X,y]，进行网格搜索
    grid = grid.fit(X, y)

    # 返回网格搜索后的最优模型
    return grid.best_estimator_

模型健壮性¶

一个最优的模型不一定是一个健壮模型。有的时候模型会过于复杂或者过于简单，以致于难以泛化新增添的数据；有的时候模型采用的学习算法并不适用于特定的数据结构；有的时候样本本身可能有太多噪点或样本过少，使得模型无法准确地预测目标变量。这些情况下我们会说模型是欠拟合的。

问题 13 - 实用性探讨¶

简单地讨论一下你建构的模型能否在现实世界中使用？

提示：回答以下几个问题，并给出相应结论的理由：

  • 数据中呈现的特征是否足够描述一个房屋？
  • 在波士顿这样的大都市采集的数据，能否应用在其它乡镇地区？
  • 你觉得仅仅凭房屋所在社区的环境来判断房屋价值合理吗？

问题 13 - 回答：¶

对我们所建立模型的实用性探讨如下： 1、数据是在数十年跨度范围内所采集的，在数十年内，很多与房价相关的条件，如社会环境，家庭结构，大人的思想观念，社区的年龄结构等会发生剧烈的变化，不同时间段内，人们在选择房屋时所考虑的因素可能已经发生了很大的改变，这使得早期的数据对于今天的影响已经不是很大。 2、数据中仅仅只有3个特征：房间数、社区低收入阶层比例和学生/老师数量比例。仅仅从常识来判断，一个房屋的价格至少会与以下主要因素有关：房屋面积、交通便捷程度、周边的配套服务环境（商店，医院，学校）、周边治安、当地经济发展水平等等。所给数据特征只覆盖了上述因素的一部分（且大都是以间接的方式。除了房间数量与房屋面积有较大关联外，其它特征与主要因素的关联度并不高）。因此，所给的数据特征并不完备。 3、数据是在波士顿这样的大城市采集的，在乡镇（乃至小城市）地区提供房价预测时，不能非常依靠这份数据训练出来的模型。 4、仅仅凭借房屋所在社区环境来判断房屋价值忽略了一个重要的外部因素：周边经济的发展状况和交通条件。居民选择住房首先要考虑自己的工作地点。在经济发达、工作机会丰富的地区，旺盛的购房/租房需求会推高房价。
